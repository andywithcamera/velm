# Contributing

Velm is open to issues, discussions, and pull requests, but contributions that are intended to be merged into the project are accepted only under the contributor terms in [CLA.md](CLA.md).

## Before You Open a Pull Request

- Open an issue or discussion first for significant changes.
- Keep changes focused and well scoped.
- Include tests where the change affects runtime behavior.
- Disclose any third-party code, assets, or text included in your contribution.

## Contributor Agreement

Before a contribution is merged, the maintainer requires a signed copy of [CLA.md](CLA.md). You will be prompted to sign via a bot comment on your pull request.

If you are contributing code written in the course of employment, education, or contract work, make sure you have the approvals needed to sign the agreement.

## Licensing Model

The public repository is licensed under `AGPL-3.0-only`. The project is also intended to support alternative commercial licensing by the Project Owner. If that is not acceptable for your contribution, do not submit it for inclusion.
